## Architecture

```
Data (train.csv)
  │
  ├─→ EDA (shape, dtypes, target distribution, missing values)
  │
  ├─→ Feature Engineering
  │     ├─ RFM: Recency (days since last txn), Frequency (txn count), Monetary (avg balance)
  │     ├─ Demographics: age, gender, geography (one-hot)
  │     ├─ Product holdings: count, types
  │     ├─ Temporal: trend deltas, product decay, channel shift
  │     └─ Behavioral: complaints, service calls, tenure, complaint escalation
  │
  ├─→ Train/Val Split (StratifiedKFold, 5-fold, shuffle=True, random_state=42)
  │
  ├─→ Model Ensemble (LightGBM + XGBoost + CatBoost)
  │     ├─ Default params → baseline CV score per model
  │     ├─ Bayesian tuning → optimized CV score per model
  │     └─ Average predictions → final ensemble score
  │
  ├─→ Threshold Optimization (on CV folds, not train set)
  │
  ├─→ Business Translation
  │     ├─ LTV derivation (from balance, products, tenure)
  │     ├─ Segment mapping (probability → risk tier)
  │     └─ Retention playbook (action per segment, estimated $ impact)
  │
  ├─→ Presentation (notebook narrative + summary)
  │
  └─→ Dashboard (Streamlit)
        ├─ Customer drill-down (select → risk score + SHAP)
        ├─ Segment overview (churn distribution by tier)
        ├─ Retention playbook (actions + $ impact)
        └─ SHAP force plot (why this customer?)
```

## Decisions Made During Review

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Data leakage | Full-data feature engineering | Hackathon test set is separate file; mild risk acceptable for speed |
| D2 | LTV source | Derive from dataset features | More defensible to judges than industry benchmarks |
| D3 | Reproducibility | Add requirements.txt | Judges need to run the notebook |
| D4 | CV strategy | StratifiedKFold(5, shuffle=True) | Preserves churn ratio in each fold |
| D5 | Inline assertions | Add to TODOS.md | Prevents silent failures in time-constrained hackathon |
| D6 | Dashboard | Full interactive Streamlit dashboard | Judges remember what they can click; rare in hackathons |
| D7 | Model ensemble | LightGBM + XGBoost + CatBoost | Ensemble gains 0.005-0.02 AUC over single model |
| D8 | Temporal features | Trend deltas, product decay, channel shift | Behavioral shifts capture churn signals that snapshots miss |

## Failure Modes

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Data leakage via RFM | Medium | High — inflated CV, poor leaderboard | D1: acceptable risk for hackathon |
| Silent NaN in features | Medium | Medium — model fails silently | Inline assertions (D5) |
| Memory overflow | Low | High — notebook crashes | dtype optimization if needed |
| Single-class CV fold | Low | Medium — metric undefined | StratifiedKFold prevents this |

### Key Interactions to Verify
- Judge runs notebook top-to-bottom without errors
- All cells produce output (no silent failures)
- Plots render correctly
- Model converges (no warnings)

### Edge Cases
- Missing values in features → handle with imputation or LightGBM native
- Zero-variance features → drop before training
- Imbalanced target → class_weight='balanced' + StratifiedKFold
- Infinite values from division → replace with NaN, then impute

### Critical Paths
- Data loads correctly → features engineered → model trains → metrics reported → business translation complete
